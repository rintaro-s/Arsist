# Arsist アーキテクチャ - ARアプリの構造

## ARアプリとは何か

ARアプリは「現実世界にデジタル情報を重ねる」アプリケーションです。

```
┌─────────────────────────────────────────────────────────────┐
│                    ユーザーの視界                              │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │                                                         │  │
│  │     ┌─────┐                                             │  │
│  │     │ HP  │  ← UI/HUD（視界に固定）                       │  │
│  │     │ 100 │                                             │  │
│  │     └─────┘                                             │  │
│  │                                                         │  │
│  │              🏠  ← 3Dオブジェクト（現実世界に配置）         │  │
│  │             /│\                                         │  │
│  │            / │ \    現実の床                              │  │
│  │  ─────────────────────────────────────                  │  │
│  │                                                         │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## 3つのレイヤー

### 1. 3Dシーン（Scene）- 空間に配置するもの

| 特性 | 説明 |
|------|------|
| **何** | 現実世界に「置く」3Dオブジェクト |
| **動き** | ユーザーが移動しても位置は変わらない |
| **例** | 家具、キャラクター、建物、アート作品 |
| **トラッキング** | 6DoF（位置+回転を追跡） |

**使用場面:**
- ARインテリア: 家具を部屋に仮配置
- ARゲーム: モンスターが床に出現
- 建築: 建物の3Dモデルを現地で確認

### 2. UI/HUD - 視界に固定するもの

| 特性 | 説明 |
|------|------|
| **何** | ユーザーの視界に「貼り付く」2D要素 |
| **動き** | ユーザーの視線に追従する |
| **例** | メニュー、スコア表示、通知、ボタン |
| **トラッキング** | 3DoF/Head-locked（回転のみ追跡） |

**使用場面:**
- 操作UI: メニュー、設定画面
- 情報表示: 時刻、通知、ステータス
- ゲームHUD: HP、スコア、ミニマップ

### 3. ロジック（Logic）- 動作を定義するもの

| 特性 | 説明 |
|------|------|
| **何** | 3DとUIを連携させる処理 |
| **例** | ボタンを押したら3Dオブジェクトが動く |

## 3DとUIの関係図

```
┌────────────────────────────────────────────────────────────────┐
│                        Arsistプロジェクト                        │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ┌──────────────────┐         ┌──────────────────┐             │
│  │    3Dシーン        │         │     UI/HUD       │             │
│  │  (Scene View)     │         │   (UI View)      │             │
│  ├──────────────────┤         ├──────────────────┤             │
│  │ - Cube            │         │ - StartButton    │             │

│  │ - Light           │  連携    │ - MenuPanel      │             │
│  └──────────────────┘         └──────────────────┘             │
│          │                            │                        │
│          │                            │                        │
│          ▼                            ▼                        │
│  ┌──────────────────────────────────────────────────┐          │
│  │                   ロジック                         │          │
│  │  OnButtonClick("Start") → Show3DObject("Char")  │          │

│                              │                                 │
│                              ▼                                 │
│  ┌──────────────────────────────────────────────────┐          │
```

## プレゼンテーションモード

| モード | 3D | UI | 用途 |
|--------|----|----|------|
| **World Anchored (6DoF)** | ✅ 空間固定 | △ 控えめ | ARゲーム、インテリア |
| **Floating Screen (3DoF)** | △ 控えめ | ✅ 追従パネル | 動画視聴、ブラウザ |
| **Head-locked HUD** | ✗ なし | ✅ 完全固定 | ダッシュボード、通知 |

## 実装詳細

### MainCamera（必須）

ARアプリには**必ずMainCameraが必要**です：

```csharp
// Unity側で自動生成される
XREAL_Rig/
  XR Origin/
    Camera Offset/
      Main Camera          // これがないとARは動かない
        - Camera component
        - AudioListener
        - TrackedPoseDriver // 頭の動きを追跡
```

### UI配置の仕組み

```csharp
// Head-locked/3DoFの場合
Canvas.renderMode = WorldSpace;
Canvas.position = Camera.main.forward * distance;  // カメラ前方に配置
Canvas.rotation = Camera.main.rotation;            // カメラと同じ向き

// 6DoFの場合
Canvas.position = worldPosition;  // 世界座標で固定
```

## よくある構成例

### 例1: ARインテリアアプリ
```
3D: 家具モデル（ソファ、テーブル）
UI: 家具選択メニュー、配置ボタン、削除ボタン
ロジック: メニューから選択→床にタップ→家具配置
```

### 例2: ARゲーム
```
3D: 敵キャラクター、アイテム、エフェクト
UI: HP、スコア、攻撃ボタン
ロジック: 敵に視線→攻撃ボタン→ダメージ→スコア加算
```

### 例3: AR情報表示
```
3D: なし（または控えめな装飾）
UI: 天気、時刻、通知、カレンダー
ロジック: 定期更新、スワイプで切替
```

## Arsistでの作り方

1. **新規プロジェクト作成** → テンプレート選択
2. **Scene View** → 3Dオブジェクトを配置
3. **UI View** → UI要素を配置
4. **Logic View** → イベント接続（将来）
5. **Build** → APK生成 → XREAL Oneで実行
